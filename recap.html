01 · CMS

Multi-project CMS foundation — projects sheet → live registry → landing page → blank-SOT scaffold

Sajangnim was hardcoded into the dashboard. Now it's just one row in a Google Sheet your producers can edit. Adding a new show is one click — no engineer in the loop.
  • Master Projects sheet — 12-column source of truth (slug, title, type, status, bible_sheet_id, drive_folder_id, parent_show, owner, created, script_url, shotlist_status, notes). Pre-seeded with sajangnim.
  • Dynamic registry — read_projects() reads the sheet on every request (60s cache), auto-discovers episode sheets from each show's Drive folder by name pattern. Old hardcoded GALLERY_REGISTRY retired.
  • /projects landing page — grid view of all projects with type chips (Series / POC / Concept / Client), status badges, episode counts, owner emails. Filter chips at top.
  • /api/new-project endpoint — accepts title + type + slug, runs _create_blank_sot.py in a background thread, appends a master row, flushes the cache. Producers will fire this from a modal in CMS commit 5.
  • + Add cover — click the dashed placeholder on any project card → file picker → upload → 16:9 cover image lives at <show>/cover.{jpg,png,webp} on Drive, anyone-with-link.

02 · VIDGEN

Vidgen identity faithfulness + no more orphan zombies

Five real bugs in one batched commit. The chars-don't-match issue from this afternoon, plus the false "watcher gave up after 90 polls" red banners, plus the storyboard jobs that kept getting orphaned by my own deploys.
  • Char-grouped refs — all PARK MIN-JUN refs bunched, all TARA refs bunched. Sort: image (attire/hair) → video (face) → audio (voice). Deterministic per click.
  • Hair binding — image binding now says "wardrobe, attire, AND HAIR come from the still X reference image" (was just "wardrobe and attire"). Closes the hair-drift gap.
  • Subprocess detach — start_new_session=True on every Popen. Render redeploys SIGTERM gunicorn but the running storyboard / vidgen subprocess survives and completes its sheet writeback. No more "stuck on Generating" zombies.
  • Watcher pagination — gallery JS now polls /debug/jobs?n=200&only=all. Was hitting the default only=failed which filtered out successful jobs entirely — that's why every successful gen showed "watcher gave up after 90 polls". Real cause, finally found.
  • 480p / 15s / 9:16 default — dropped from 1080p. ~62% cheaper per V1+V2 click while iterating. CLI flag still works for 1080p delivery cuts.

03 · AUTH

Google OAuth live · door is locked

No more anonymous access. No one can hit your APIs to drain BytePlus credits.

06 · RELIABILITY

502 → 200 across all 6 episodes

Episodes 2-6 had been bouncing 500s for hours this morning. Root-caused twice; fixed properly the second time.
  • Bibles route to ep01 sheet — eps 2-6 don't have their own CHARACTERS / LOCATIONS / etc. tabs (those live at series level). Build was crashing with KeyError: 'CHARACTERS'.
